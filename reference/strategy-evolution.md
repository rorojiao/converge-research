# 策略自进化规则

## 每轮评估维度

Phase 4 对本轮进行四维评估：

### 1. 完成度增量
本轮各子目标完成度相比上轮的提升。用于判断是否在向目标逼近。

**评分约束（与 convergence.md 校准规则联动）：**
- 首轮（round=1）：单个子目标完成度 ≤ 60%
- 每轮增量：单个子目标 ≤ 30%
- 评分前必须列出该子目标"仍未回答的具体问题"（至少 2 个）作为扣分依据
- 违反以上规则的评分无效，需重新评估

### 2. 信息增量（调研模式）
本轮发现的新信息占累积知识库的比例。递减说明接近饱和。

**首轮特殊处理：**
- 首轮 info_delta 强制为 100%（分母为零），不参与收敛判断

### 3. 指标改进（优化模式）
本轮实验的指标变化幅度。

### 4. 策略有效性
本轮采用的搜索方向/优化方向是否产出高价值结果。

## 进化规则

基于评估结果，按以下规则更新 STRATEGY.md：

### 规则 1：放大有效方向
若某方向在本轮产出 ≥2 个高价值发现 → 下轮加深该方向（更具体的关键词、更深的子问题）

### 规则 2：淘汰无效方向
若某方向连续 2 轮无产出 → 移入 EXHAUSTED.md，下轮不再尝试

### 规则 3：发散探索
若本轮所有方向增量都低 → 引入全新搜索角度（换视角、换关键词体系、换信息源类型）

### 规则 4：聚焦短板
哪个子目标完成度最低 → 下轮优先分配资源给该子目标

### 规则 5：元策略调整
若连续 2 轮增量递减 → 改变搜索策略类型（如从广度搜索切换到深度挖掘，或从论文搜索切换到代码仓库搜索）
