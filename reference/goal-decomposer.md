# 目标分解协议

## 调研模式

将用户目标分解为 3-7 个子目标。每个子目标包含：

```markdown
### 子目标 N: [名称]
- **描述**: 需要回答的具体问题
- **完成标准**: 什么情况下算完成（可判断的描述）
- **权重**: 0.X（所有权重之和 = 1.0）
- **验证方法**: LLM自评（基于知识库内容判断是否充分回答）
- **当前完成度**: 0%
```

分解原则：
1. 子目标之间不重叠（MECE原则）
2. 覆盖目标的所有关键方面
3. 权重反映对最终目标的重要程度
4. 完成标准必须具体可判断

## 优化模式

将优化目标分解为可量化的子维度：

```markdown
### 子目标 N: [优化维度]
- **描述**: 具体优化什么
- **指标**: 从 metric 命令输出中提取哪个值
- **基线值**: 当前值（首轮运行 metric 获取）
- **目标值**: 期望达到的值（可选，不设则持续优化）
- **权重**: 0.X
- **当前值**: [基线值]
```

## 初始 STRATEGY.md 生成

基于子目标，为第一轮生成初始调研/优化策略：
- 调研模式：列出 3-5 个搜索方向 + 关键词 + 预期信息源
- 优化模式：列出 3-5 个优化方向 + 预期改进机制
