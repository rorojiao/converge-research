# 收敛判断算法

## 前置保护

### 最小轮次保护
```
min_rounds: 3  # 前3轮跳过所有收敛检查，直接进入下一轮
```
当 `round < min_rounds` 时，**不做任何收敛判断**，直接执行"未收敛时的处理"。

## 多维收敛条件

每轮结束时（Phase 5），且 `round ≥ min_rounds` 后，检查以下终止条件，**满足任一即收敛**：

### 条件 1：目标完成度达标
```
加权总完成度 = Σ(子目标完成度 × 权重) ≥ 85%
```

### 条件 2：信息饱和（调研模式）
```
连续 2 轮信息增量 < 10%
信息增量 = (本轮新发现数 / 累积总发现数) × 100%
```
含义：已经很难找到新信息了，继续搜索收益递减。

### 条件 3：优化停滞（优化模式）
```
连续 3 轮无指标改进
```
含义：已经找不到可优化的空间了。

### 条件 4：安全上限
```
当前轮次 ≥ 最大轮次（默认10）
```

## LLM 自评分校准规则

**此规则强制适用于所有子目标完成度评分，防止分数膨胀导致过早收敛。**

### 规则 A：首轮评分上限
```
round == 1 时，任何单个子目标完成度 ≤ 60%
```
原因：一轮广度搜索不可能深入理解任何子目标，60% 已是极限。

### 规则 B：每轮增量上限
```
任何单个子目标每轮增量 ≤ 30%
```
即：如果上轮某子目标是 40%，本轮最高只能评为 70%。

### 规则 C：扣分依据强制列出
```
每次评分时，必须先列出该子目标"仍未回答的具体问题"（至少 2 个），
然后基于这些未回答问题的严重程度决定扣分幅度。
不列出未回答问题 → 该子目标完成度不得超过 50%。
```

### 规则 D：首轮信息增量特殊处理
```
round == 1 时：
  - 累积总发现数 = 0（分母为零），强制 info_delta = 100%
  - 首轮 info_delta 不参与收敛判断，不记入 consecutive_low_delta
```

## STATE.md 中的收敛信号追踪

```markdown
## Convergence Signals
- info_delta_history: [35%, 22%, 15%, 9%, 7%]
- consecutive_low_delta: 2        # 连续低增量轮数
- stall_count: 0                   # 连续无改进轮数（优化模式）
- weighted_completion: 78.5%       # 加权总完成度
- min_rounds: 3                    # 最小轮次保护
```

## 收敛时的处理

1. 读取 KNOWLEDGE.md（调研）或 RESULTS.tsv（优化）
2. 读取 reference/output-format.md
3. 生成 FINAL_REPORT.md
4. 更新 STATE.md: `status: converged`
5. 向用户输出收敛摘要：
   - 总轮次
   - 最终完成度
   - 收敛原因
   - 报告文件路径

## 未收敛时的处理

1. 更新 STATE.md: `round: N+1`
2. 更新 info_delta_history（首轮记录 100% 但标注为不参与收敛判断）
3. 输出本轮摘要（简短）：当前完成度、本轮增量、下轮方向预告
4. 等待下次触发
